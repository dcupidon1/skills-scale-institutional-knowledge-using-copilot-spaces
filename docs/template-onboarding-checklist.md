## Week 1: Getting Started

### Day 1 - Welcome & Setup

#### Administrative
- [ ] HR paperwork completed
- [ ] Welcome email received with key information
- [ ] Manager/buddy introduction meeting scheduled
- [ ] Team introduction meeting scheduled
- [ ] Building access/badges obtained
- [ ] Parking/transportation information received

#### Technology Setup
- [ ] Laptop/workstation received and set up
- [ ] Email account activated and tested
- [ ] Slack/communication tools access granted
- [ ] VPN and security tools installed
- [ ] Password manager set up
- [ ] GitHub/version control access granted
- [ ] Development environment set up (IDE, tools)
- [ ] Access to necessary applications (Figma, Jira, etc.)

#### Initial Meetings
- [ ] Welcome meeting with manager (30 min)
- [ ] Team introduction and overview (1 hour)
- [ ] Buddy system explained and buddy assigned
- [ ] First 1:1 with manager scheduled

### Days 2-5 - Orientation & Learning

#### Company & Team Knowledge
- [ ] Company mission, values, and culture overview
- [ ] Organization chart and team structure reviewed
- [ ] Team charter and goals discussed
- [ ] Read: Company handbook
- [ ] Read: Team documentation in docs/ folder
- [ ] Review: OctoAcme Project Management Overview
- [ ] Review: Role-specific persona documentation

#### Process & Workflows
- [ ] Project management process overview
- [ ] Development workflow and branching strategy explained
- [ ] PR and code review process covered
- [ ] CI/CD pipeline overview
- [ ] Release and deployment process reviewed
- [ ] Communication norms and meeting cadence explained
- [ ] Time tracking and reporting process (if applicable)

#### Tools & Systems Training
- [ ] GitHub Projects board walkthrough
- [ ] Documentation repository tour
- [ ] Monitoring and logging tools introduction
- [ ] Bug tracking system training
- [ ] Knowledge base/wiki access and navigation

## Week 2: Getting Involved

### Learning & Shadowing
- [ ] Shadow team members in daily standup
- [ ] Observe sprint planning meeting
- [ ] Attend team demo/showcase
- [ ] Shadow pair programming session (if applicable)
- [ ] Observe customer interaction or user research session
- [ ] Review recent retrospective notes

### First Contributions
- [ ] First small task/ticket assigned
- [ ] Create first pull request or deliverable
- [ ] Participate in code/design review
- [ ] Ask questions and get comfortable with team
- [ ] Contribute to team meeting discussion

## Week 3-4: Increasing Ownership

### Project Work
- [ ] Assigned to current sprint/project work
- [ ] Participating fully in team ceremonies
- [ ] Taking on progressively larger tasks
- [ ] Contributing ideas and feedback
- [ ] Pair with team members on complex work

### Relationships & Network
- [ ] 1:1 meetings with key cross-functional partners
- [ ] Introduced to stakeholders
- [ ] Added to relevant Slack channels and distribution lists
- [ ] Understand who to go to for different questions
- [ ] Building rapport with immediate team

### Process Participation
- [ ] Attended sprint retrospective
- [ ] Contributed feedback in retrospective
- [ ] Participated in sprint planning
- [ ] Helping refine backlog or requirements
